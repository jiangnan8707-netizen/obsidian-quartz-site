# 个人知识库模式

这份笔记把“用 LLM 维护个人知识库”的思路整理成可执行的本地结构。

## 核心想法

传统 RAG 更像是“问答时临时检索”，每次提问都要重新拼装信息。这里采用的是另一种方式：让 LLM 持续维护一个不断增厚的 Markdown 知识库。

当新资料进入时，LLM 不只是索引它，而是会：

- 阅读并提炼关键信息
- 更新相关页面
- 补充交叉引用
- 标记新旧信息冲突
- 把高质量的分析结果留在库里

这样，知识不是每次都重新推导，而是会不断累积。

## 三层结构

### 1. 原始资料

原始资料区，只放来源文件，不直接改写内容。

### 2. Wiki

由 LLM 维护的 Markdown 页面集合，包含摘要、实体页、概念页、对比页和总览页。

### 3. Schema

给 LLM 的操作说明文件，比如 `AGENTS.md`，用来约束结构、流程和维护方式。

## 常见操作

### 导入

加入一份新资料后，读取、总结、更新索引、更新相关页面，并写入日志。

### 查询

提问时先读索引，再读相关页面，最后给出带引用的综合答案。好的答案也可以重新写回知识库。

### 检查

定期检查：

- 页面之间是否有矛盾
- 是否有过时内容
- 是否有孤立页面
- 是否缺少交叉引用
- 是否需要补充新资料
