# Codex 介绍

Codex 是一个面向软件开发和本地工作流的 AI 助手。它可以阅读项目文件、理解代码结构、修改文件、运行本地命令，并根据结果继续推进任务，适合用来处理开发、调试、文档整理和自动化操作。

## 能做什么

- 阅读和分析本地项目代码
- 修改、创建或整理文件
- 运行测试、构建命令和常用检查
- 帮助定位报错原因并给出修复方案
- 编写说明文档、脚本和工作流
- 协助处理表格、PDF、网页和本地应用相关任务

## 适合的使用场景

### 代码开发

Codex 可以根据需求修改项目中的代码，并尽量遵循现有项目风格。它会先查看相关文件，再决定如何改动，适合处理功能开发、缺陷修复、代码重构和测试补充。

### 项目维护

Codex 可以帮助检查仓库状态、整理文档、生成 README、分析依赖问题，也可以协助处理构建失败、测试失败和配置错误。

### 本地自动化

在允许的环境中，Codex 可以运行本地命令或控制部分桌面应用，用于重复性操作、文件处理、数据整理和流程自动化。

## 工作方式

Codex 通常会按以下方式完成任务：

1. 先查看项目结构和相关文件
2. 判断最小可行改动范围
3. 修改文件或运行必要命令
4. 验证结果，例如运行测试或检查生成文件
5. 总结完成内容和剩余风险

## 使用建议

- 尽量说明目标，而不只是说明要改哪个文件
- 如果有固定格式、语言或输出路径，直接写清楚
- 如果希望只给方案、不修改文件，需要提前说明
- 对重要改动，可以要求 Codex 先做代码审查或列出计划

## 示例指令

```text
帮我修复这个项目的登录报错，并运行相关测试。
```

```text
帮我新建一个 README，介绍项目用途、安装方式和启动步骤。
```

```text
检查当前仓库有没有明显的问题，只做 review，不要修改代码。
```

## 注意事项

Codex 可以提高处理本地项目和重复任务的效率，但它仍需要明确的任务边界。对于涉及生产环境、隐私数据、删除文件、发布上线等高风险操作，最好先确认操作范围和备份状态。
